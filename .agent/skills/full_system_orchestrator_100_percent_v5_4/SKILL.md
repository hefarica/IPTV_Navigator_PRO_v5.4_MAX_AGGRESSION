## 🏗️ 2. ARQUITECTURA DE INTEGRACIÓN MULTI-NIVEL

Para orquestar el 100% del ecosistema **sin romper el Strict 1:1 EOF Protocol**, los flujos se han encapsulado utilizando inyección paramétrica:

### Capa A: Inyección de `Global Headers` (God-Tier)
Las señales de orquestación de más alto nivel que rigen el comportamiento holístico del reproductor se integraron en la función `generateGlobalHeader`. Al declarar estos Tags debajo del `#EXTM3U` principal, se forza a OTT Navigator, TiviMate y VLC a acatar el "Player Enslavement Protocol" **antes** de procesar siquiera el primer canal:

```m3u8
#EXT-X-APE-NET-TOLERANCE:BDP=100MB|CWND=64|RTT_MAX=3000ms
#EXT-X-APE-QUANTUM-IMMORTALITY:v10-ENABLED
#EXT-X-APE-TELEMETRY-SYNC:NETFLIX-GRADE
#EXT-X-APE-EVASION-GET:405-REWRITE-TO-HEAD
#EXT-X-APE-FALLBACK-HEURISTICS:LATENCY<100ms
#EXT-X-APE-SYNC-SLAVE:MASTER_ORIGIN_SYNC_V2
```

**Por qué funciona:** Al situarlo globalmente, garantizas que cualquier reproductor que cachee el M3U8 localice preventivamente estas capacidades de evasión multihilo.

### Capa B: Inyección `Strict 1:1 URL Query` (Fijación Blindada de Canal)
La orquestación moderna requiere que el backend/worker reciba instrucciones precisas *por canal*. Sin embargo, el uso incorrecto de `#EXT-X-STREAM-INF` (exclusivo para listas máster) estaba rompiendo la lectura de canales directos (VOD o Live).

El "Full System Orchestrator" establece el paradigma **URL Parameter Binding**, inyectando los comandos de evasión, forcings de CMAF, y tolerancias directamente a la cola de la URL en la función `generateChannelEntry` (línea 3125+):

// http://nov.../14.m3u8?ape_cmaf_force=true&ape_quantum_sync=v10&evasion_405=head&latency_target=100&sync_slave=master_origin_v2
