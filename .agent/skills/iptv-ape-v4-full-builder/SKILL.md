# IPTV OMEGA ABSOLUTO Builder

Esta skill encapsula el **Estado del Arte** del streaming personalizado. Transforma cualquier ecosistema M3U8 tradicional en una arquitectura OMEGA híbrida, reduciendo el peso de las listas en un 95% mientras multiplica el control sobre el reproductor mediante orquestación JSON en tiempo real.

## Arquitectura OMEGA

La arquitectura se compone de 2 piezas maestras que trabajan en sincronía:

1. **Generador Polimórfico (`omega_list_generator.py`)**: 
   - Comprime 5,272 directivas en un Payload JSON de 4 líneas.
   - Aplica 1% de uniqueness (hashes dinámicos) para evasión de firmas.
   - Clasifica el contenido en 5 Perfiles Maestros (Deportes, Cine, Noticias, etc.).

2. **Motor de Reconstrucción (`resolve_quality_OMEGA.php`)**:
   - Intercepta el Payload JSON y lo expande a 359 líneas de código dictatorial por canal en tiempo real.
   - Inyecta directivas en TODOS los formatos simultáneamente (`#EXTHTTP`, `#KODIPROP`, `#EXTVLCOPT`, `#EXT-X-APE`).

## Las 5 Doctrinas (Conocimiento PhD)

El ecosistema OMEGA está construido sobre 5 disciplinas técnicas:

1. **Player Enslavement**: Forzado de Hardware Decode y deshabilitación de ABR.
2. **GPU Rendering**: LCEVC Phase 4, HDR10+ (5000 nits) y AI Super Resolution.
3. **Evasión de ISP**: Swarm Phantom Hydra Stealth (hasta 2048 conexiones TCP).
4. **Polimorfismo e Idempotencia**: Hashes MD5/SHA256 dinámicos con Session IDs estables.
5. **Telchemy TVQM**: Monitoreo de calidad bidireccional y resolución de fallos en <60ms.

## Workflow de Implementación (De 0 a 100%)

Para implementar OMEGA en un sistema existente, debes seguir estrictamente los prompts operativos incluidos:

1. **Inyección de Conocimiento**: Lee y asimila `references/PROMPT_SKILLS_AGENTE_PHD.md`.
2. **Actualización del Generador**: Sigue paso a paso `references/PROMPT_OPERATIVO_IMPLEMENTACION.md` para modificar el generador M3U8 actual.
3. **Actualización del Resolver**: Sigue paso a paso `references/PROMPT_OPERATIVO_RESOLVER.md` para construir el `resolve_quality_unified.php` v5.0.
4. **Auditoría**: Verifica la implementación contra `references/CHECKLIST_DIRECTIVAS_OMEGA.md`.

## Recursos Incluidos

- **`scripts/omega_list_generator.py`**: El generador maestro de referencia.
- **`scripts/resolve_quality_OMEGA.php`**: El motor de reconstrucción v5.0.
- **`scripts/ape_anatomy_compare.py`**: Herramienta de análisis forense multi-versión.
- **`templates/OMEGA_EXAMPLE_LIST.m3u8`**: Lista de ejemplo generada correctamente.
- **`references/`**: Toda la documentación técnica, prompts operativos y checklists.

## Reglas de Oro

- **NUNCA optimizar líneas**: Si una directiva parece redundante, no lo es. El control dictatorial requiere redundancia.
- **Cadena de Degradación Inviolable**: Los 7 niveles (CMAF → fMP4 → TS → HTTP Redirect) deben inyectarse siempre.
- **Idempotencia Visual**: El resultado en pantalla debe ser matemáticamente idéntico en cada zapping, aunque los hashes de la lista muten.
