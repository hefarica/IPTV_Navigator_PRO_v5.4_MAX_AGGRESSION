# Workflow Maestro: Implementación de Ecosistema OMEGA Absoluto

Usa este workflow cuando debas actualizar el entorno IPTV actual al estándar híbrido extremo OMEGA (0 cortes, 0 errores, Evasión total).

## Pasos Universales

1. **Inyección de Conocimiento Periférico**
   * El agente siempre debe procesar los scripts y configuraciones leídos contra las 5 disciplinas técnicas maestras dictadas en `iptv-ape-v4-full-builder`.
   * Verifica que las librerías base (PHP 8.1+, cURL, etc) están funcionales en el entorno del host o despliegue.

2. **Refactorización del Frontend UI M3U8 Generator (`m3u8-typed-arrays-ultimate.js`)**
   * Obligar la inyección de `OMNI-ORCHESTRATOR-V5-OMEGA`.
   * Usar el modelo de extrapolación JSON donde `#EXTHTTP` embeba el `{"paradigm":"OMNI-ORCHESTRATOR-V5-OMEGA", ...}` realocado.
   * Modificar agresivamente `#EXTM3U` agregando el `#X-OMEGA-TIMESTAMP` dinámico de JS nativo.

3. **Inyección Directa en Resolver (`resolve_quality_unified.php`)**
   * Integrar el OMEGA Reconstructor dentro de la "Médula Híbrida".
   * Asegurar 357 líneas de expansión, 100% libre de errores fatales 500 y con constantes bien establecidas (Ej: `RQ_MAX_NITS`, `RQ_MAX_BW`).
   * Purgar cualquier `ENDPOINT PRINCIPAL` doble que asfixie el renderizado PHP y rompa el FPM/NGINX socket thread a M3U8.

4. **Despliegue y Validación VPS**
   * Actualizar `/var/www/` rutas base (html, iptv-ape, m3u8) estrictamente.
   * Lanzar una prueba de cURL (Ejemplo: `curl -s -k "https://178.156.147.234/resolve_quality_unified.php?omega_test=1"`).
   * Validar 3 aserciones críticas: Idempotencia en el payload (Sesión igual, Nonce diferente), factor de control extremo 60000ms al hardware, y HTTP 200 OK persistente en todo zaping L7.
