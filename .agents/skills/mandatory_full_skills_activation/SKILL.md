# Mandatory Full Skills Activation Protocol (MFSAP)

## REGLA NÚCLEO — INMUTABLE E INVIOLABLE

**TODO** trabajo realizado por el agente Antigravity DEBE incorporar la consulta y aplicación de TODAS las skills relevantes del ecosistema. No existe excepción. No existe "solo aplico las 3 más relevantes". Se aplica el **100% del conocimiento disponible**.

## Protocolo de Activación Obligatoria

### 1. Al inicio de CADA tarea, el agente DEBE:

1. **Escanear** el directorio `.agent/skills/` para identificar TODAS las skills disponibles
2. **Clasificar** las skills en 3 categorías de aplicabilidad:
   - 🔴 **DIRECTAS**: Skills que se aplican directamente a la tarea (ej: `resolve-architecture` al editar `resolve_quality.php`)
   - 🟡 **INDIRECTAS**: Skills que aportan restricciones o validaciones (ej: `anti_509_blind_resolution` al hacer cualquier cambio de backend)
   - 🟢 **CONTEXTUALES**: Skills que aportan buenas prácticas generales (ej: `competitive_advantage_execution_protocol`)

3. **Leer** las skills DIRECTAS antes de ejecutar
4. **Consultar** las skills INDIRECTAS para validar que no se violen restricciones
5. **Aplicar** las doctrinas de las skills CONTEXTUALES como guía de calidad

### 2. Al EDITAR código del backend (PHP):
- SIEMPRE consultar: `resolve-architecture`, `resolver_blindaje_total`, `anti_509_blind_resolution`
- SIEMPRE verificar contra: `protocolo_herencia_estricta`, `functional_implementation_guarantee`
- SIEMPRE validar orden M3U8 contra: `arquitectura_anatomica_m3u8_ultimate`, `m3u8_120_120_perfection_invariant`

### 3. Al EDITAR código del frontend (JS):
- SIEMPRE consultar: `cableado_dinamico_omni_inyeccion`, `generacion_lista_ape_typed_arrays_ultimate`
- SIEMPRE verificar contra: `compliance_indexeddb_architect`, `protocolo_persistencia_absoluta`

### 4. Al GENERAR listas M3U8:
- SIEMPRE aplicar: `generacion_m3u8_bulletproof_1_to_1`, `pipeline_blindaje_m3u8`
- SIEMPRE validar: `balanced_scorecard_m3u8`, `auditoria_forense_linea_por_linea`

### 5. Al DESPLEGAR al VPS:
- SIEMPRE seguir: `despliegue_quirurgico_vps`, `ape_deployment_safety_guardrails`
- SIEMPRE verificar: `anti_509_blind_resolution`, `resolver_blindaje_total`

## Consecuencias de Violación

Si el agente omite una skill relevante y esto causa un bug, la PRIMERA acción al diagnosticar el bug DEBE ser declarar: "Este bug se habría evitado si hubiera consultado la skill [X]".

## Excepción Única

La ÚNICA excepción es cuando el usuario pide explícitamente una tarea trivial (ej: "dime la hora", "formatea esta tabla"). En ese caso, no se requiere activación de skills.
